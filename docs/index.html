O IntegrOn mantém a lista de CNPJs no arquivo YAML config_cnpj_integration.yaml e o valor do ponteiro NSU no arquivo <CNPJ>.txt. Esses arquivos são lidos a cada ciclo de integração — qualquer alteração entra em vigor automaticamente no próximo ciclo CRON, sem reiniciar o serviço.

Arquivos do Sistema

ArquivoLocalização padrãoFunção
config_cnpj_integration.yaml C:\integron\ Lista de contas CNPJ — certificado, modo batch e identificação
<CNPJ>.txt C:\integron\nsu\ Ponteiro NSU por CNPJ — último NSU processado com sucesso

Pré-requisitos

⚠️
Antes de começar Toda operação de CNPJ requer acesso de administrador à VM e as credenciais do Azure fornecidas pelo cliente.
Acesso necessário
Administrador à VM Windows — PowerShell como Administrador
Certificado digital
Nome do certificado no Azure Key Vault — fornecido pelo cliente
Credencial Azure
Permissão Key Vault Secrets Officer ou superior
CNPJ válido
14 dígitos com dígito verificador correto — validar antes de configurar
1

Adicionar um Novo CNPJ

Siga os passos em ordem. Não é necessário reiniciar o serviço.

1

Adicionar o CNPJ ao arquivo de configuração

Edite o arquivo C:\integron\config_cnpj_integration.yaml e acrescente uma nova entrada na lista accounts:

YAML — config_cnpj_integration.yaml
accounts:
  # CNPJs existentes...
  - cnpj: "12345678000195"
    certificateName: "nfse-cert-empresa-a"
    batchQuery: true

  # ← Novo CNPJ adicionado aqui
  - cnpj: "98765432000100"
    certificateName: "nfse-cert-empresa-nova"
    batchQuery: true
CampoRegra
cnpj14 dígitos numéricos. Deve passar na validação de dígito verificador. Deve ser único na lista.
certificateNameNome exato do segredo no Azure Key Vault
batchQueryQuando false, a API retornará apenas um elemento; com true poderá retornar uma lista de NFS-e
2

Criar o arquivo de ponteiro NSU

Cada CNPJ precisa de um arquivo <CNPJ>.txt no diretório C:\integron\nsu\ com o NSU inicial. Execute no PowerShell como Administrador:

PowerShell — como Administrador
# Substitua 98765432000100 pelo CNPJ real (somente dígitos)
"0" | Set-Content -Path "C:\integron\nsu\98765432000100.txt" -Encoding UTF8
💡
NSU inicial diferente de zero? Se o CNPJ já foi processado anteriormente em outro sistema, informe o último NSU conhecido para evitar re-download de documentos já coletados. Substitua "0" pelo valor correto.
3

Confirmar no próximo ciclo CRON

✅
Sem reinicialização O IntegrOn detectará o novo CNPJ automaticamente no próximo ciclo CRON. Nenhuma espera ou reinicialização é necessária.
2

Alterar Configuração de um CNPJ

Atualize campos como certificado ou modo batch sem interromper o serviço.

1

Identificar o CNPJ a alterar

Abra C:\integron\config_cnpj_integration.yaml e localize a entrada pelo valor do campo cnpj.

2

Fazer a alteração desejada

Trocar o certificado digital:

  1. Faça upload do novo certificado no Azure Key Vault.
  2. Atualize o campo certificateName no YAML com o nome do novo segredo.

Alterar o modo batch:

YAML — alteração de batchQuery
  - cnpj: "12345678000195"
    certificateName: "nfse-cert-empresa-a"
    batchQuery: false  # alterado de true para false
⚠️
Não altere o campo cnpj O CNPJ é a chave de identificação. Para trocar o CNPJ, remova a entrada antiga e adicione uma nova seguindo o fluxo de Adicionar CNPJ.
3

Confirmar no próximo ciclo CRON

✅
A alteração entra em vigor automaticamente no próximo ciclo CRON. Nenhuma reinicialização é necessária.
3

Remover um CNPJ

Desative a coleta de NFS-e para um CNPJ sem apagar histórico.

⚠️
Preserve o arquivo de ponteiro NSU Antes de remover o arquivo <CNPJ>.txt, confirme se a coleta foi concluída. O arquivo registra o último NSU processado — se o CNPJ for reativado no futuro, ele poderá retomar de onde parou.
1

Remover a entrada do arquivo de configuração

Edite C:\integron\config_cnpj_integration.yaml e delete a entrada completa do CNPJ:

YAML — antes da remoção
accounts:
  - cnpj: "12345678000195"  # ← manter
    certificateName: "nfse-cert-empresa-a"
    batchQuery: true

  - cnpj: "98765432000100"  # ← remover este bloco
    certificateName: "nfse-cert-empresa-b"
    batchQuery: true
YAML — após a remoção
accounts:
  - cnpj: "12345678000195"
    certificateName: "nfse-cert-empresa-a"
    batchQuery: true
🚫
Nunca esvazie a lista O arquivo deve sempre conter ao menos um CNPJ. Uma lista vazia causa falha na inicialização (STEP_02_ACCOUNT_LIST_LOADED).
2

Confirmar no próximo ciclo CRON

✅
O CNPJ deixará de ser processado a partir do próximo ciclo CRON. Nenhuma reinicialização é necessária.
3

(Opcional) Arquivar o arquivo de ponteiro NSU

Recomenda-se manter o arquivo <CNPJ>.txt em C:\integron\nsu\. Se desejado, mova-o para um diretório de arquivo em vez de deletar:

PowerShell — mover para arquivo
# Mover para arquivo (não deletar)
New-Item -ItemType Directory -Path "C:\integron\nsu\arquivados" -Force
Move-Item "C:\integron\nsu\98765432000100.txt" "C:\integron\nsu\arquivados\"
4

Redefinir o Ponteiro NSU

Reinicie ou ajuste o ponto de coleta de um CNPJ específico.

🚨
Ação irreversível sem backup Redefinir o NSU para 0 fará o serviço baixar novamente todos os documentos desde o início. Isso pode gerar tráfego e custo de storage desnecessários (o IntegrOn sobrescreve arquivos existentes, portanto não haverá dados corrompidos).
1

Verificar o NSU atual

PowerShell
# Ver o NSU atual do CNPJ
Get-Content "C:\integron\nsu\12345678000195.txt"
# Exemplo de saída: 1000423
2

Aguardar o ciclo CRON não estar em execução

O IntegrOn possui um overlap guard: se um ciclo estiver ativo, o próximo é descartado. Para alterar o NSU com segurança, aguarde a conclusão do ciclo atual verificando o log:

PowerShell — verificar estado do ciclo
# Aguardar "Integration completed" ou "Integration started" recente
Get-Content "C:\integron\logs\integron.log" -Tail 20 |
  Where-Object { $_ -match "Integration" }
3

Definir o novo valor de NSU

PowerShell — redefinir NSU
# Redefinir para zero (reiniciar coleta do início)
"0" | Set-Content -Path "C:\integron\nsu\12345678000195.txt" -Encoding UTF8

# OU definir para um NSU específico (retomar de um ponto conhecido)
"1000000" | Set-Content -Path "C:\integron\nsu\12345678000195.txt" -Encoding UTF8
ℹ️
Formato do arquivo O arquivo deve conter apenas um número inteiro não-negativo. Qualquer outro conteúdo causa falha no passo STEP_04_NSU_POINTER_READ e gera um incidente.

Verificar o Resultado

Após qualquer operação, confirme que está funcionando corretamente.

1

Monitorar o log após a próxima execução do CRON

PowerShell — filtrar por CNPJ
Get-Content "C:\integron\logs\integron.log" -Tail 50 |
  Where-Object { $_ -match "98765432000100" }
2

Sinais de sucesso esperados no log

NívelmessageSignificado
INFO Integration started CNPJ foi reconhecido e o ciclo iniciou
INFO Integration completed Documentos encontrados e gravados no Blob Storage
WARN Mensagem de nenhum documento localizado Normal — API retornou HTTP 404. Nenhum incidente gerado.
3

Verificar o ponteiro NSU atualizado

PowerShell
# Deve conter um número maior que zero após coleta bem-sucedida
Get-Content "C:\integron\nsu\98765432000100.txt"
4

Verificar arquivos no Azure Blob Storage

No Portal Azure → Conta de Armazenamento → Contêineres → verifique se há uma pasta com o número do CNPJ contendo arquivos no formato <ChaveAcesso>_<NSU>.XML.

Resolução de Problemas

Erros mais comuns após operações de CNPJ e como resolvê-los.

Erro no logPassoCausa provávelSolução
Arquivo <CNPJ>.txt ausente ou valor não-numérico STEP_04 Arquivo de ponteiro NSU não foi criado ou está corrompido Criar o arquivo com "0" (ver Passo 2 de Adicionar CNPJ)
ClientAuthenticationException STEP_05 Client secret expirado ou variáveis de ambiente ausentes Verificar variáveis VaultTenantId, VaultClientId, VaultClientSecret
Failed to retrieve certificate '...' STEP_05 Nome do certificado no certificateName não coincide com o segredo no Key Vault Verificar o nome exato no Portal Azure → Key Vault → Segredos e corrigir o YAML
CredentialUnavailableException STEP_05 Service Principal sem acesso ao Key Vault do CNPJ Atribuir o papel Key Vault Secrets User ao Service Principal
StorageException STEP_17 Service Principal sem acesso ao Blob Storage Atribuir o papel Storage Blob Data Contributor
CNPJ não aparece nos logs — Erro de sintaxe no YAML ou CNPJ com dígito verificador inválido Validar o YAML com um linter; verificar se o CNPJ possui 14 dígitos válidos
Lista de contas vazia — falha na inicialização STEP_02 Todos os CNPJs foram removidos do YAML Manter ao menos uma entrada no arquivo config_cnpj_integration.yaml
PowerShell — filtrar logs por CNPJ
Get-Content "C:\integron\logs\integron.log" |
  Where-Object { $_ -match "98765432000100" }
🔁
Recuperação automática Após corrigir a causa raiz, não é necessário reiniciar o serviço. O próximo ciclo CRON retomará automaticamente a partir do último NSU registrado.
